# 原理

首先了解一下GitLab Pages运行的原理。与GitHub不同的是，GitLab需要上传一个 .gitlab-ci.yml 的文件，同时生成的项目文件必须要到 /public 目录中，见详情。

## .gitlab-ci.yml

image: node:9.11.1

pages:
 cache:
   paths:
   - node_modules/

 script:
 - npm install
 - npm run docs:build
 artifacts:
   paths:
   - public
 only:
 - master

valine 的饮用必须放到mounted中，否则无法编译，报错，window is undefined
